# Compte rendu

Aujourd’hui, j’ai commencé par évaluer les projets ESP.

Ensuite, je suis revenu sur notre projet pour remettre en place les tests du frontend. J’ai récupéré les anciens fichiers Cypress, corrigé la configuration, et adapté les tests au nouveau système d’authentification. J’ai également configuré un fichier `cypress.env.json` pour éviter d’inclure des identifiants ou mots de passe directement dans le dépôt.

Après ça, j’ai tenté de relancer complètement le backend avec Docker. Le frontend fonctionnait correctement, mais le backend a commencé à planter à cause des migrations. La base PostgreSQL contenait déjà des tables (notamment `AspNetRoles`), donc même après avoir supprimé les migrations et recréé une nouvelle initiale, la migration échouait avec l’erreur **"relation already exists"**.

En clair : le volume Docker garde une base déjà existante et tant qu’on ne la réinitialise pas complètement, le backend refusera de démarrer proprement. Résultat : Swagger ne se lance même plus tant que les migrations plantent.

J’ai aussi appris que pour accéder à Swagger, il faut d’abord se connecter via `/api/login` avec l’admin (`admin@orange.fr` + le mot de passe dans le `.env` backend), récupérer le token, puis l’insérer dans le bouton **Authorize** en haut à droite, préfixé par `Bearer`. Mais comme les migrations échouent, impossible d’accéder à Swagger pour le moment.
